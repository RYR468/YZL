# 益志领 · 导师发展与赋能体系（YL Mentor System）

> 本文是系统级别/激励的设计依据，对齐基金会正式《导师发展与赋能体系》方案。
> 系统已实现：综合评估分自动评定 L1-L3 级别 + 荣誉证书；退出机制、年会、成就看板等运营动作为流程说明（系统以字段记录状态）。

## 一、核心目标与原则
激活精英导师资源，形成高效运作的社群，为学生提供持续、高质量的帮扶，实现公益价值与导师职业价值的结合。
原则：**结构化管理 · 多元化激励 · 动态化管理 · 社群化链接**。

## 二、导师级别（L1-L3）
| 级别 | 名称 | 核心要求 | 激励与特权 |
|---|---|---|---|
| **L1** | 新锐导师 | 完成培训；每年至少服务 4 小时 | 认证证书、基础项目对接权 |
| **L2** | 资深导师 | 连续服务 1 年；每年 >10 小时；参与重点项目 | 优先对接优质生源、受邀核心活动、企业合作项目名额（如拜尔斯道夫项目） |
| **L3** | 首席导师 | 连续服务 3 年；每年 >20 小时；卓越贡献（如设计课程） | 基金会顾委成员、官方专访宣传、**致信 CEO 表彰** |

## 三、评估方法（系统自动算级别）
**综合评估分（0-100）= 服务时长 40% + 服务质量 40% + 深度贡献 20%**
- **服务时长**：累计志愿服务小时数（满分基准 200h）
- **服务质量**：学生反馈平均分 1-10（满分 10）
- **深度贡献**：拉新导师 / 企业赞助 / 课程开发 / 治理建议（满分基准 50 分）

定级（演示阈值，运营可调，见 [config/incentive.json](../config/incentive.json)）：
- **L3 首席**：综合 ≥ 80 且深度贡献 ≥ 30
- **L2 资深**：综合 ≥ 60 且有深度贡献
- **L1 新锐**：其余有服务记录者

## 四、激励体系（组合激励）
1. **精神荣誉与官方认可**：年度服务证书（系统可生成，列明贡献与时长）、"杰出导师"奖项、官网/公众号"导师风采"专访专栏
2. **机构认可——致信 CEO 机制**：对 L3 或年度杰出导师，基金会撰写《导师社会影响力总结报告》，正式致信其所在机构 CEO/高管（系统在 L3 证书自动标注）
3. **学生反馈与成就感**：成就看板、感谢卡、影响力故事卡、年度影响力故事评选
4. **社交价值与物质关怀**：年度导师大会、年底定制小礼品、导师私享会

## 五、动态管理（评估与退出机制）
评估标准权重：**服务时长 40% · 服务质量 40% · 深度贡献 20%**（与级别评估一致）。
退出机制：
- **预警**：服务不达标提醒
- **停牌**：连续两年零服务
- **移除**：三年失联 或 严重投诉

（系统以「服务状态」字段记录：**活跃 / 停牌 / 移除**）

## 六、导师信息登记表（字段）
| 类别 | 字段 | 用途 |
|---|---|---|
| 基本信息 | 姓名、年龄段、性别、联系方式 | 身份识别与日常沟通 |
| 职业信息 | 所在机构（公司全称）、职位/职称、行业/专业 | 致信 CEO 所需、分类匹配 |
| 服务信息 | 导师级别(L1-L3)、累计服务时长、指导学生总数、对接学生记录 | 分级管理、核心量化 |
| 评估激励 | 学生反馈平均分(1-10)、年度影响力案例、是否同意致信 CEO | 服务质量、宣传 |
| 管理信息 | 注册日期、状态（活跃/停牌/移除） | 动态管理 |

## 七、系统落点
- [config/incentive.json](../config/incentive.json)：评估权重 + 级别阈值（运营可调）
- [matching/incentive.py](../matching/incentive.py)：`compute_score`（综合评估分）/ `compute_level`（L1-L3）
- `/certificate/<导师ID>`：荣誉证书（L3 自动含「致信 CEO」标注）
- 导师管理视图：级别列 + ❓ 悬停评定标准
